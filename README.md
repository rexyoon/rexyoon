자기소개
->
저는 새로운 기술이나 문제를 마주했을 때, 정답을 빠르게 찾기보다 원인을 이해하고 구조적으로 해결하려는 태도를 중요하게 생각해왔습니다. 이러한 성향은 전공 수업과 개인 프로젝트, 그리고 협업 경험을 통해 더욱 확고해졌습니다.

대학 생활 중 기초 이해도와 실무 역량의 부족함을 느껴 이를 보완하고자 1년간 휴학 후 부산에서 풀스택 개발 부트캠프에 참여했습니다. 최종 팀 프로젝트에서는 비전공자 팀원들로 구성된 팀에서 협업을 진행했으며, 이해 수준 차이로 인한 소통 문제를 개인의 역량 문제가 아닌 팀 전체의 과제로 인식했습니다. 이 과정에서 특정한 소통 문제로 인해 프로젝트 일정이 지연되었고, 이를 해결하기 위해 서로의 부족한 부분을 공유하고 설명할 수 있도록 함께 학습하며 공통된 이해를 만드는 방식을 선택했습니다. 저는 설명과 의견 조율의 역할을 맡으며, 팀 내에서 협업의 핵심은 속도가 아닌 이해의 정렬임을 배웠습니다.

이후 Spring Boot 기반 개인 프로젝트를 진행하며, 기술 스택을 확장하기 위해 기존에 사용해보지 않았던 도커를 개발 환경 구성에 적용했습니다. 실행 환경을 단순히 분리하는 것에 그치지 않고, 컨테이너 간 역할과 의존성을 이해하며 환경을 구성해 나갔습니다. 또한, 프론트엔드에서는 Vite 기반의 개발 환경을 직접 설정하여 빌드 구조와 개발 생산성의 차이를 비교하고 이해하려 노력했습니다.

이 과정에서 OpenAI API 요청 제한 오류를 겪으며 요청 흐름과 서버 로직을 점검해 중복 호출이 원인임을 확인했습니다. 이를 바탕으로 호출 구조를 분리하고 캐싱 전략을 적용하여 문제를 해결했습니다. Spring AI 설정 과정에서는 Elasticsearch와 PgVector 간 빈 충돌 문제를 기술 역할 분석과 선택적 구성을 통해 해결하며, 문제의 구조를 이해하고 판단하는 개발자로 성장할 수 있었습니다.

이러한 경험들을 통해 저는 기술적 문제뿐 아니라 팀 내 의견 충돌 상황에서도 서로의 관점을 존중하며 가장 합리적인 방향을 찾는 태도의 중요성을 배웠습니다. 앞으로도 익숙함에 머무르지 않고, AI 분야의 최신 기술을 주도적으로 학습하며, 문제의 본질을 함께 이해하고 해결하는 개발자로 성장하겠습니다.
